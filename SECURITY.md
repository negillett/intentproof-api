# Security policy

Repository: [IntentProof/intentproof-api](https://github.com/IntentProof/intentproof-api).

## Supported versions

Security updates are applied to the latest release line on the default branch.

## Reporting a vulnerability

Please do **not** file undisclosed security issues as public GitHub issues.

Use [GitHub private vulnerability reporting](https://github.com/IntentProof/intentproof-api/security/advisories/new) for this repository (or your organization's equivalent process if the repo is forked).

Include: affected version or commit, reproduction steps, and impact assessment if available.

## Scope

This policy covers the IntentProof API service in this repository and its build/release workflows. Deployment-specific concerns (runtime secrets handling, network policy, cloud IAM, and operator key management) remain environment responsibilities for the deploying team.
